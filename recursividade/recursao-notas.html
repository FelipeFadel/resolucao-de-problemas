Dicas para Programar em TypeScript (Recursão)

1. Sempre defina o caso base

Toda função recursiva precisa de um caso de parada. Sem isso o código entra em loop infinito. 

if (n === 0) {
  return 0;
}

Pergunte sempre: “Quando minha recursão deve parar?”

2. Garanta progresso na recursão

A cada chamada recursiva o problema precisa ficar menor.

return soma(n - 1);

Se o valor não diminuir ou mudar, a recursão nunca termina.

3. Pense no problema menor

Recursão funciona quando você resolve um problema usando uma versão menor dele. 

n! = n * (n-1)!

4. Teste primeiro casos pequenos

0
1
2
3

Isso ajuda a encontrar erros no caso base.

5. Use nomes claros

Evite:

x
y
z

Prefira:

current
index
total
remaining

6. Cuidado com Stack Overflow

Recursões muito profundas podem quebrar o programa.

  • O problema realmente diminui?
  • Existe caso base?

7. Use parâmetros auxiliares

function soma(n: number, total: number = 0)

Isso ajuda a guardar o estado da execução.

8. Simule no papel

funcao(3)
funcao(2)
funcao(1)
funcao(0)

Isso ajuda a entender o fluxo da recursão.

9. Evite loops se a questão pedir recursão

for
while

Use chamadas da própria função.

10. Verifique os tipos no TypeScript

function factorial(n: number): number

Isso evita erros e ajuda o compilador.

Regra de Ouro da Recursão

  1. Caso base
  2. Problema menor
  3. Chamada recursiva correta

Se essas três coisas estiverem corretas, a recursão vai funcionar.
